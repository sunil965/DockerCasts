Data Aggregation and Grouping in Pandas

1. What You'll Learn in This Section
In this lesson, you'll learn to:

Explain what data aggregation and grouping are, and why they work together to summarise large datasets.
Identify which columns belong in groupby and which belong in aggregation functions, using the categorical vs. numerical distinction.
Apply groupby, agg, and multi-column grouping to answer real-world analytical questions in Pandas.
Clean up grouped output with reset_index, sort_values, and rename to produce publication-ready summary tables.

2. Detailed Explanation
What data aggregation is — and why it matters
Data aggregation is the process of combining many individual data values into a single summary number. Think of it like squeezing a long spreadsheet down to its essentials — instead of staring at 1,000 rows, you see one row per group with the total, average, or count.

Raw data, even after cleaning, can contain thousands of rows. Aggregation compresses those rows into a compact table of summary statistics that is easy to interpret. Patterns that hide inside thousands of rows become visible immediately once you aggregate. For example, knowing that 50% of company revenue goes to salaries informs next-year budgeting — a finance team might then allocate 55% to cover salary increments and new hires.

Scale matters here too. Machine learning algorithms need large datasets to generalise well. Large Language Models such as ChatGPT and Gemini perform well precisely because they were trained on enormous volumes of data — effectively the full publicly crawled web. Any supervised model benefits from having more labelled examples. Aggregation is one of the key tools that lets you make sense of data at that scale.

Aggregation also existed before Python — Microsoft Excel supports SUM, AVERAGE, COUNT, MIN, and MAX on columns, and many analytical decisions were made with Excel alone for years. Pandas offers two meaningful advantages over Excel:

Integration with programming. An aggregated result becomes a Python value that feeds directly into subsequent logic — conditionals, loops, and machine learning pipelines. Excel does not support this kind of programmatic chaining.
Ease of application and visualisation. A single Pandas command produces the aggregated output, and visualising that output is equally concise.
Grouping — the partner operation
Grouping is the operation of partitioning a dataset into subsets based on the values in one or more categorical columns, so that an aggregation function can then run independently on each subset.

The key insight is this: grouping alone is not useful — it makes sense only when combined with an aggregation function. The two always operate together.

Here is a concrete example. Imagine a dataset of 1,000 employees with columns employee, city, and salary. Grouping by city splits the rows into four buckets — Mumbai, Delhi, Hyderabad, and Bengaluru. Applying mean to the salary column within each bucket produces a four-row output table: one average salary per city. That single table answers the question "which city pays the most?" in seconds.

Categorical vs. numerical variables — knowing which column goes where
Before writing any groupby code, you need to classify each column correctly.

Variable type	Definition	Examples from the employee dataset	Role
Categorical	Takes a fixed, finite set of named values	city, department, gender	Goes inside groupby(...)
Discrete (numerical)	Takes countable numeric values	salary, experience	Target for aggregation functions
Applying a statistical function like mean to a categorical column (for example, computing the mean of city) is logically invalid — there is no sensible numeric average of city names. Always group on categorical columns and aggregate on numerical ones.

The employee dataset — the running example
All the code examples below work on a 10-row, 6-column DataFrame built from a Python list-of-lists and converted with pd.DataFrame(...):

import pandas as pd

data = [
    ['EMP001', 'HR',        'Bengaluru', 'Male',   45000, 2],
    ['EMP002', 'IT',        'Hyderabad', 'Female', 70000, 5],
    ['EMP003', 'Finance',   'Pune',      'Male',   62000, 4],
    ['EMP004', 'IT',        'Bengaluru', 'Male',   76000, 7],
    ['EMP005', 'Marketing', 'Hyderabad', 'Female', 54000, 3],
    # ... (10 rows total)
]
df = pd.DataFrame(
    data,
    columns=['employee', 'department', 'city', 'gender', 'salary', 'experience']
)
Inspect it with:

df.head()            # First 5 rows
df.shape             # (10, 6)
list(df.columns)     # ['employee', 'department', 'city', 'gender', 'salary', 'experience']
df.info()            # Column types and non-null counts
df.info() confirms 10 non-null entries per column — no missing values to worry about.

Core aggregation functions
All five functions are called on a grouped DataFrame object produced by groupby. They are valid only on numerical columns.

Function	What it produces
sum()	Total of all values within each group
mean()	Average value within each group
count()	Number of rows within each group
min()	Smallest value within each group
max()	Largest value within each group
groupby — the foundation of grouped aggregation
groupby is the Pandas function that partitions a DataFrame into named groups. The syntax has three distinct parts:

df.groupby('department')['salary'].mean()
'department' inside groupby(...) — the categorical column that defines the groups.
['salary'] in square brackets — the numerical column to aggregate.
.mean() — the aggregation function to apply.
The distinction between the parentheses and the square brackets is important: the column inside groupby(...) is the grouping key; the column inside [...] is the aggregation target.

Running this on the employee dataset produces:

department
Finance      65000.0
HR           46500.0
IT           72666.7
Marketing    52000.0
Notice the output is float64 (Pandas promotes int64 to float64 because mean can produce fractions). Convert back to integers with .astype(int):

df.groupby('department')['salary'].mean().astype(int)
You can swap in any of the five core functions:

df.groupby('department')['salary'].count()
# Finance: 3, HR: 2, IT: 3, Marketing: 2

df.groupby('department')['salary'].sum()
# Finance: 195000, HR: 93000, IT: 218000, Marketing: 104000

df.groupby('department')['salary'].min()
# Finance: 62000, HR: 45000, IT: 70000, Marketing: 50000

df.groupby('department')['salary'].max()
# Finance: 68000, HR: 48000, IT: 76000, Marketing: 54000
Watch out for: placing the aggregation target column inside groupby(...) by mistake. df.groupby('salary')['department']... groups by a numerical column and tries to aggregate a categorical one — both are backwards.

agg — multiple statistics in one table
Running four separate commands to get count, mean, min, and max is tedious. agg computes multiple statistics at once. Pass a list of function names to create one table with all results:

df.groupby('department')['salary'].agg(['count', 'mean', 'min', 'max'])
department	count	mean	min	max
Finance	3	65000.0	62000	68000
HR	2	46500.0	45000	48000
IT	3	72666.7	70000	76000
Marketing	2	52000.0	50000	54000
You can also apply the same single function to multiple numerical columns at once using double square brackets:

df.groupby('department')[['salary', 'experience']].mean()
This works when the same function applies uniformly to both columns. If you need different functions for different columns (for example, sum on salary but count on experience), you would use separate commands.

Multi-column grouping — revealing sub-group patterns
To see summaries broken down by two categorical dimensions at once, pass a list of column names to groupby:

df.groupby(['department', 'gender'])['salary'].mean()
This creates one output row per unique combination of department and gender. With four departments and two genders, you could have up to eight rows. Our dataset has only six (two combinations missing):

department	gender	salary (mean)
Finance	Female	65000.0
Finance	Male	65000.0
HR	Male	46500.0
IT	Female	70000.0
IT	Male	74000.0
Marketing	Female	52000.0
This reveals a salary difference within the IT department: male employees earn an average of 74,000 while female employees earn 70,000 in this dataset. A single-column groupby by department alone would have hidden this.

You can group by city with the same approach:

df.groupby('city')['salary'].mean()
# Bengaluru: 60000, Hyderabad: 65000, Pune: 58000
Hyderabad has the highest average salary; Pune the lowest.

reset_index — turning group labels back into columns
When groupby produces a summary, the grouping column(s) become the DataFrame's index rather than regular columns, and the default 0-based row numbers disappear. This can make the result look odd and harder to work with downstream.

reset_index converts index labels back to regular columns and adds row numbers:

department_summary = df.groupby('department')['salary'].mean().reset_index()
# Before: department names are the index, no row numbers
# After:  numeric index 0–3, two columns — department and salary
It works equally well on multi-column grouped results:

dept_gender_summary = (
    df.groupby(['department', 'gender'])['salary']
    .mean()
    .reset_index()
)
# Produces 6 rows with columns: index, department, gender, salary
sort_values and rename — finishing touches
Because reset_index returns a regular DataFrame, you can chain any standard Pandas operation onto it. sort_values sorts the result to show the highest-paying department first:

department_summary.sort_values('salary', ascending=False)
The numeric index assigned by reset_index stays associated with its original row — it does not re-number after sorting.

One remaining issue: the aggregated column still carries the original name salary, even though it now contains average salaries. rename fixes this by accepting a dictionary mapping old names to new ones:

department_summary = department_summary.rename(
    columns={'salary': 'average_salary'}
)
After this, department_summary has two clearly labelled columns: department and average_salary. Only the salary column is renamed; all other columns are untouched.

Real-world applications and the conceptual flow
Aggregation and grouping answer high-level business questions that individual records cannot:

Domain	Typical question
Sales data	Which city generated the highest total sales?
Employee data	What is the average salary per department?
Education data	What are the average, min, and max marks per course?
Healthcare data	Which hospital treated the most patients today?
The healthcare example illustrates how aggregated output feeds directly into downstream code. Apply groupby('hospital_name') + count() on a treatment column to get patient counts per hospital, then use a Python max lookup to identify the winner — no machine learning needed, just aggregation chained with simple logic.

The overall flow every analysis follows:

Start with raw data — a large DataFrame where conclusions are hard to see by inspection.
Choose an aggregation goal — which summary statistic answers your question?
Identify your grouping column(s) — categorical variables that define your subgroups.
Run groupby + aggregation function (or agg for multiple statistics).
Clean the output — reset_index, sort_values, rename as needed.
Feed the result into downstream Python logic for decisions.
The sales dataset (columns: order_id, city, product_category, quantity, sales_amount) follows exactly this pattern. To find which city generated the highest total sales, apply groupby('city')['sales_amount'].sum() and identify the maximum — the same approach used on the employee dataset, just with different column names.

3. Key Takeaways
Aggregation compresses large datasets into compact summary statistics (sum, mean, count, min, max). It turns thousands of rows into a handful of insight-rich numbers.
Grouping always pairs with aggregation — groupby partitions the data by a categorical column so the aggregation function runs independently on each subset.
Always group on categorical columns (city, department, gender) and aggregate on numerical columns (salary, experience). Mixing these up is the most common mistake.
agg(['count', 'mean', 'min', 'max']) produces multiple statistics in one table. Multi-column grouping (groupby(['col1', 'col2'])) reveals interactions between two categorical dimensions.
reset_index, sort_values, and rename turn a raw grouped result into a clean, human-readable DataFrame that feeds naturally into downstream Python logic.
Mental model: Think of groupby + aggregation as a sorting tray. You drop all your data into the tray, and each compartment (group) catches the rows that belong to it — then you weigh (aggregate) each compartment separately to get one number per slot.
